# 正文写作内核

本文件只覆盖采矿工程、岩石力学和 CO₂ 地质封存论文的 S8–S12。模型负责语义推理和成文，脚本负责材料快照、字段闭环和确定性检查。任何流畅表达都不能替代证据。

## 1. 写作入口

S9 通过后运行：

```text
python scripts/build_manuscript_context.py --vault <论文库>
python scripts/manuscript_guard.py plan --vault <论文库>
```

只有两条命令均返回可继续状态时才进入正文。`manuscript_context.json` 是本轮写作材料快照，记录路径、哈希、结构和必要内容；原始账本仍是权威来源。快照过期时重建，不直接手改哈希。

## 2. 论证主线

全稿应能压缩为：

`工程或科学问题 → 已知规律及其条件 → 未解决缺口 → 本研究动作 → 决定性证据 → 机制解释或工程含义 → 适用边界`

每个核心主张都应回答：

- 对象：煤、岩石、矿体、围岩、盖层、储层、裂隙、井筒或充填体；
- 尺度：矿物/孔隙、试样、巷道/井筒、储层/矿区或区域；
- 条件：应力路径、温压、含水或盐度、气体组成、加载/注入速率和时间尺度；
- 证据类型：实验、理论、数值模拟、现场监测、反演、统计关联或文献综合；
- 结果：方向、效应量或范围、不确定性；
- 边界：不能外推到何种岩性、尺度、工况或时间范围。

严禁无证据转换：实验相关性→现场因果性、二维或理想化模拟→场地验证、短期试验→长期封存安全、不同地质或工程场景之间的无条件类比。

## 3. 章节合同

### 引言

从工程风险或科学问题收敛到一个可检验缺口。公平描述已有研究，创新必须由对照文献和本研究证据界定。最后一段说明研究动作、证据路线和边界，不预支未证实的结果。

### 方法

使同行能够复现：样品/场地来源与状态、设备或模型、边界和初始条件、加载或注入路径、测量与统计、参数来源、校准/验证、网格与时间步、软件和版本、排除规则。不得使用“常规方法”“标准条件”“进行了统计分析”等无操作信息的替代句。

### 结果

每个小节按“问题/试验 → 观察 → 定量证据 → 图表定位 → 小结”组织。结果报告发生了什么，不用机制语言掩盖缺失验证。异常值、离散性、失败场景和反例不得静默删除。

### 讨论

解释结果为何成立、还有哪些替代解释、与既有研究为何一致或冲突。必须处理岩性和尺度效应、THMC 条件、实验与现场差异、模型假设、敏感性和不确定性。关联证据只能支持关联；长期稳定性必须有对应时间尺度证据。

### 结论

逐条回答 RQ，给出决定性证据和适用条件。不得加入新结果、新机制或新引用。避免把“可行性迹象”写成工程许可或长期安全证明。

## 4. 逐章工作单

每章开写前建立一个短工作单：

```yaml
chapter_id: ""
purpose: ""
promises_to_fulfil: []
claim_ids: []
evidence_ids: []
figure_table_ids: []
method_details_required: []
word_budget: null
forbidden_overclaims: []
open_evidence_gaps: []
```

章后不做空泛自评分，只检查：承诺是否完成、每个关键主张是否有来源定位、图表是否真的支持文字、方法是否可复现、尺度/因果/时间边界是否越界、数字和术语是否与上游一致。缺证据时保留明确占位并阻塞该主张，不用顺畅文字补空白。

## 5. 语言处理

语言调整顺序为：科学逻辑 → 段落任务 → 句法 → 词汇。目标期刊与用户样文决定拼写、节奏和标点，不采用通用固定句长。

重点清理：空泛重要性、宣传性形容词、模糊归因、机械三段式、重复转折、名词堆叠、主体消失的被动句、结果与讨论混写、模板化结尾。不得为了“更像真人”加入观点、虚构细节或改变学术语域。

改写后运行：

```text
python scripts/manuscript_guard.py invariants \
  --before <改写前.md> --after <改写后.md> --vault <论文库>
```

默认阻塞数字/单位、DOI、引用、图表/公式号、追溯 ID、方向词和主张强度变化。只有人工核实同义替换不改变科学含义时，才可使用 `--allow-language-token-changes`，并将决定写入决策账本。

## 6. 投稿前完整性

S12 运行：

```text
python scripts/manuscript_guard.py integrity --vault <论文库>
```

检查范围包括：未清占位、题名一致性、正文引用与参考文献闭环、DOI 重复、证据和引用映射状态、图件路径与 QA、机制图占位、未关闭 P0/P1。脚本返回 `invalid` 时不得进入 S13；语义审阅仍需定位最强反方解释和领域攻击面，不能只依赖格式检查。

## 7. 领域审阅攻击面

至少检查：试样代表性与各向异性；含水/饱和状态；应力路径和加载速率；温压、盐度及 CO₂ 组成；重复性与统计单位；本构参数来源；网格和时间步收敛；校准与独立验证；实验/模拟向现场迁移；短期结果向长期安全外推；断层、裂隙、井筒和盖层泄漏路径；敏感性、不确定性和失败场景。

高优先级问题必须给出位置、影响、修复动作和复验条件。没有位置或证据的“看起来没问题”不构成验收。
