# 质量门

## 每阶段五项

1. 完整性：必需文件、字段和稳定 ID 存在。
2. 可追溯性：主张回到全文锚点、数据、脚本输出或用户确认事实。
3. 一致性：题名、RQ、术语、数字、单位、符号、图号、引文键和版本一致。
4. 边界性：结论没有跨越证据类型、对象、尺度、条件、时间或相关/因果边界。
5. 真实性：验证状态来自脚本或逐项证据，不来自模型自评分或人工预填 `passed`。

## 一票否决

- 未确认路径即创建论文库，或覆盖用户已有内容。
- 未经用户确认即开始 Mode B、替用户分级或超过批量上限。
- 自动下载失败却未展示清单、等待通知和验证 PDF 就进入 S5。
- 把摘要、网页、元数据或检索片段当成全文精读或正文证据。
- 编造文献、DOI、数据、实验、公式参数、统计量、图件、结果、作者信息或仓储编号。
- 用短期试验声称长期封存稳定，用模拟声称现场验证，用相关性声称机制，或无条件跨岩性/尺度类比。
- 润色改变数字、单位、方向、否定、限定语、图号、引文键或证据强度。
- 图件登记表写 `passed`，但缺源数据、矢量/PNG、QA 报告、统计披露或正文引用。
- 运行时缺失仍声称已生成 PDF、完成 OCR、投稿级绘图或期刊政策核验。
- S12/S13 仍有 P0/P1、证据缺口、未解决占位或跨文件冲突。

任一项出现时停止，写入 blocker 和整改动作；修复后重新运行确定性验证器。
