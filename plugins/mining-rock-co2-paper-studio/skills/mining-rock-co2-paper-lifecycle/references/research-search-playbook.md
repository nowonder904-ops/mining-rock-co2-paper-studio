# 开题、检索与全文工作手册（S2–S5）

## S2：把主题变成可检验问题

先展开“对象 × 尺度 × 条件 × 驱动 × 响应 × 证据路线”：

- 对象：煤、岩样、围岩、盖层、储层、裂隙、充填体或工程结构；
- 尺度：矿物/孔隙、试样、巷道/井筒、储层/矿区、区域；
- 条件：应力路径、温压、含水/盐度、气体组成与相态、加载/注入速率、时间；
- 驱动：开采、卸载、循环荷载、CO₂ 注入/浸泡、热处理、化学作用等；
- 响应：强度、变形、损伤、裂纹、渗透率、吸附解吸、注入能力、泄漏风险；
- 路线：实验、理论、数值模拟、现场监测、反演或统计关联。

形成 2–4 个候选 RQ，逐一检查：单一且可检验；方法真的能回答；数据/样品/边界条件可得；创新有对照研究；最强替代解释和预审质疑已列出。创新只允许落在新对象、新条件、新尺度、新方法、新证据或新机制解释，不用“首次、系统、全面、揭示”代替证据。

## S3：可复现检索方案

1. 从 RQ 生成 3–6 个概念块：对象、作用、响应、场景、方法和排除项；每块记录同义词、旧术语、缩写和中英文变体。
2. 默认主源为 Crossref 与 OpenAlex；与地球化学、环境或方法交叉时补 Europe PMC；预印本方法补 arXiv。Google Scholar 只作可选人工补查，出现验证码时记录并停止自动化。
3. “近两年”按执行日动态计算；经典研究单独检索，并把高被引仅视为候选代理，不等同于质量或经典认定。
4. 记录每个数据源、完整检索式、时间窗、过滤器、运行时间、结果数、原始响应路径和失败状态。
5. 当前影响因子、分区、期刊政策或作者指南必须联网核验或由用户提供；不得根据记忆声称“高分期刊”。

执行示例：

```bash
python scripts/scholarly_search.py --vault <论文库> --query <检索式> --from-year <当前年-1>
python scripts/literature_register.py import-candidates --vault <论文库> --input <论文库>/02_Search/search_candidates.csv
```

模型负责检索式、相关性、主题聚类和纳排建议；脚本负责 API 响应、来源记录、去重、稳定 ID 和 BibTeX/RIS/ENW 导出。

## 去重与筛选

优先用规范 DOI 去重；无 DOI 时使用规范题名加年份，必要时交叉核对作者和期刊。同一研究的预印本、会议版和期刊版记录 `version_relation`，不要误删为普通重复。`literature_register.csv` 是唯一长期事实源；原始检索结果只作批次证据。

筛选状态只用 candidate、priority、include、exclude；排除必须有原因。标题和摘要只能支持相关性筛选，不能进入全文证据账本。

## 合法全文获取

优先顺序：开放获取 PDF、出版社公开全文、PMC/机构仓储、作者公开稿、用户已有机构权限。运行：

```bash
python scripts/fulltext_fetch.py --vault <论文库>
```

下载器默认只尝试已登记的 OA 链接，拒绝绕过访问控制的来源；普通出版商落地页只有显式允许时才尝试。PDF 目录只能存放通过签名和 EOF 检查的 PDF；HTML/登录页进入独立缓存。任何失败都写回登记表并进入 S4.5，不能静默丢弃。

下载结果写入登记表后先验证并完成 S4，再立即运行 `download_failure_gate.py build` 进入 S4.5；这是同一工作回合的强制动作。只要失败项存在，清单生成后结束推进并等待用户。

自动取得的 PDF 初始状态为 `pdf-available`。模型实际核对首页题名、DOI（或题名加作者/期刊/年份）且无冲突后，运行 `literature_register.py verify-identity --paper-id <ID> --evidence <实际核对依据>` 才能变为 `pdf-verified`。人工补全文仍按 S4.5 的 `confirm-identity` 流程处理。

## 证据等级边界

- metadata-only：题名、作者、年份、DOI、期刊；只能用于检索和身份核验。
- abstract-lead：摘要或网页摘要；只能用于筛选和提出待核查假设。
- fulltext-anchored：带 PDF 页码、章节、图表或公式定位；可进入证据账本。
- data-anchored：带数据文件、脚本输出或用户确认事实的稳定定位；可支撑本研究结果。

S5 前必须通过人工补全文闸门。网络或数据源不可用时记录 `external_unavailable`；已取得的本地成果仍保留，但不得把部分检索说成完整覆盖。
