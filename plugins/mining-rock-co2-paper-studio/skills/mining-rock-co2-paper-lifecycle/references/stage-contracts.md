# 阶段合同

## 通用记录

每阶段记录 `stage_id`、输入文件、模型完成的语义任务、运行脚本、产物、验证结果、阻塞原因、开始/完成时间和下一阶段。聊天总结不是状态源；`pipeline_state.yaml`、决策账本和产物登记表共同构成可恢复事实。

状态只用：`not_started`、`in_progress`、`blocked_by_user`、`blocked_by_evidence`、`blocked_by_runtime`、`completed`。`blocked_by_runtime` 表示缺少网络、解析器、绘图库或转换器等环境能力，不表示插件依赖其他 SKILL。

## 前置关系

- S0 → S1：用户已确认论文库文件夹名称和父目录绝对路径。
- S1 → S2：manifest、state、任务板、决策账本、产物登记表和目录已创建或完成接管扫描。
- S2 → S3：对象、尺度、条件、驱动、响应、证据路线、可检验 RQ、创新对照和预审质疑齐全。
- S3 → S4：检索式、动态时间窗、来源、纳排规则和日志字段可复现。
- S4 → S4.5：候选文献含题名、年份、来源、DOI/稳定 ID、下载状态和失败原因；原始响应留档。
- S4.5 → S5：所有失败项在用户通知后验证为正确 PDF，或由用户明确排除并记录原因。
- S5 → S5.5：DOI/题名/版本去重、全文状态和主题分类可核查。
- S5.5 → S6：用户确认分级定义、逐篇等级、顺序、批量上限和图表要求，验证器通过。
- S6 → S7：证据账本、引用卡、笔记、索引、Wiki、Base 和 Canvas 已回写且每条可引用证据有原文定位。
- S7 → S8：研究脉络、争议和研究空白有文献映射，检索覆盖边界明确。
- S8 → S9：贡献、动机、验证路线、适用边界和不可声称事项已由用户确认。
- S9 → S10：详细骨架、章节预算、主张—证据—图表矩阵、Results—Contribution 映射和方法复现审计完成。
- S10 → S10.5：逐章门通过；数据源和每张图服务的唯一 claim ID 明确。
- S10.5 → S11：数据图有矢量、PNG/灰度预览和真实 QA；机制图提示词、登记表和正文占位双向一致。
- S11 → S12：学术润色和作者声纹校准已完成，不变量比较无未批准变化。
- S12 → S13：引用、证据、术语、图号、完整性和模拟审稿中的 P0/P1 已关闭。
- S13 → S14：交付包冻结并有哈希；只有收到真实编辑或审稿意见才进入 S14。

## 状态推进

用 `stage_control.py begin` 进入阶段。人工输入未到用 `blocked_by_user`；证据或产物不合格用 `blocked_by_evidence`；环境能力缺失用 `blocked_by_runtime`。只有 `stage_control.py complete` 实际调用阶段验证器并返回 0，才能写 completed。

S4.5 出现未解决项必须为 `blocked_by_user`；新文件出现在目录中不会自动恢复，必须先收到用户通知再验证。S5.5 的建议等级不是用户决定。S13 没有 PDF 工具时可交付 Word/LaTeX 源，但完整投稿包保持 `blocked_by_runtime`，不得降格标为完成。
